# Buckos Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build Buckos — a family behavior-currency web app where parents grant/withdraw "Buckos" for kids, with weekly allowance resets, a ledger as source of truth, and read-only kid views.

**Architecture:** Single npm package. Express + TypeScript server (`server/`) exposes a REST API and serves the built React client in production; ledger-only data model in SQLite behind a repository interface; balances/charts/resets all derived from the ledger with an injectable clock. React 18 + Vite + Tailwind v4 client (`client/`) with a CSS-variable design-token theme layer.

**Tech Stack:** Node 22, TypeScript, Express 4, better-sqlite3, cookie-session, google-auth-library, React 18, Vite 6, Tailwind CSS v4, Recharts, Vitest + Supertest, Playwright.

## Global Constraints

- `AUTH_MODE=dev` → user-picker login; `AUTH_MODE=google` → real Google OAuth. Rest of app identical in both modes.
- `PARENT_EMAILS` (comma-separated) defines parents; kid access only via a parent-created kid profile with that email; everyone else gets a friendly "You're not on the list" screen.
- All authorization enforced server-side on every route; kid hitting parent endpoint → 403.
- Ledger is source of truth; transactions immutable; balances may go negative; all amounts whole numbers.
- Weekly reset: balance → weekly allowance at configurable boundary (default Monday 00:00 server local); lazy catch-up of missed resets; each reset is a ledger entry ("Weekly reset to N") stamped at the actual boundary time.
- Env config: `PARENT_EMAILS, AUTH_MODE, GOOGLE_CLIENT_ID, GOOGLE_CLIENT_SECRET, SESSION_SECRET, DATABASE_PATH, RESET_DAY, RESET_HOUR, PORT` (+ `ENABLE_TEST_CLOCK` for e2e). Provide `.env.example`.
- Commands: `npm run dev` (single command, dev mode), `npm run build && npm start` (production), `npm run seed`, `npm test` (unit+API), `npm run test:e2e`.
- Design: Claude-like theme — cream background, dark charcoal text, warm coral/terracotta accent, serif display font (Lora) + clean sans (Inter), all tokens as CSS variables in one theme file. Mobile-first; test 375/768/1280.
- Out of scope: screen-time payoff, weekly bonuses, notifications, multi-tenancy, theme-switcher UI.

### Data model (ledger-only)

```sql
CREATE TABLE kids (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT NOT NULL,
  email TEXT NOT NULL,
  weekly_allowance INTEGER NOT NULL DEFAULT 100,
  archived INTEGER NOT NULL DEFAULT 0,
  created_at TEXT NOT NULL              -- ISO 8601
);
CREATE TABLE transactions (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  kid_id INTEGER NOT NULL REFERENCES kids(id),
  amount INTEGER NOT NULL,              -- signed delta; reset rows store (allowance - balanceBefore)
  note TEXT NOT NULL,
  type TEXT NOT NULL CHECK (type IN ('adjustment','reset')),
  actor_email TEXT,                     -- NULL for system reset entries
  created_at TEXT NOT NULL              -- ISO 8601
);
```

Balance = `SUM(amount)` over a kid's transactions. Reset rows carry the delta needed to land on the allowance, so the invariant holds and history is unambiguous ("Weekly reset to 100").

### Key interfaces

```ts
// server/types.ts
export type Role = 'parent' | 'kid';
export interface Kid { id: number; name: string; email: string; weeklyAllowance: number; archived: boolean; createdAt: string; }
export interface Txn { id: number; kidId: number; amount: number; note: string; type: 'adjustment' | 'reset'; actorEmail: string | null; createdAt: string; }
export interface SessionUser { email: string; role: Role; kidId?: number; name?: string; }

// server/repo.ts
export interface Repo {
  listKids(): Kid[];                          // non-archived
  getKid(id: number): Kid | undefined;
  findKidByEmail(email: string): Kid | undefined;   // non-archived, case-insensitive
  createKid(k: { name: string; email: string; weeklyAllowance: number; createdAt: string }): Kid;
  updateKid(id: number, patch: Partial<Pick<Kid, 'name' | 'email' | 'weeklyAllowance'>>): Kid;
  archiveKid(id: number): void;
  listTxns(kidId: number): Txn[];             // newest first
  addTxn(t: Omit<Txn, 'id'>): Txn;
  balance(kidId: number): number;             // SUM(amount)
  lastResetAt(kidId: number): string | undefined;
}

// server/clock.ts
export interface Clock { now(): Date; }

// server/ledger.ts
export function resetBoundariesSince(from: Date, now: Date, resetDay: number, resetHour: number): Date[];
export function ensureResets(repo: Repo, kid: Kid, clock: Clock, cfg: Config): void;  // lazy catch-up
export function chartData(repo: Repo, kidId: number, clock: Clock): { date: string; balance: number; isToday: boolean }[]; // 7 entries, oldest→today
```

### API contract

| Method/Path | Who | Body → Response |
|---|---|---|
| `GET /api/me` | any session | `{ user: SessionUser }` or 401 |
| `POST /api/auth/logout` | any | 204 |
| `GET /api/auth/dev-users` | dev mode only | `{ users: {email, role, name}[] }` |
| `POST /api/auth/dev-login` | dev mode only | `{ email }` → session or 403 not-on-list |
| `GET /auth/google`, `/auth/google/callback` | google mode | OAuth redirect flow; unknown email → redirect `/not-allowed` |
| `GET /api/kids` | parent | `{ kids: (Kid & { balance, chart })[] }` (runs ensureResets per kid) |
| `POST /api/kids` | parent | `{ name, email, weeklyAllowance? }` → kid; creation writes initial reset row ("Weekly reset to N") |
| `GET /api/kids/:id` | parent | `{ kid, balance, chart, transactions }` |
| `PATCH /api/kids/:id` | parent | partial `{ name?, email?, weeklyAllowance? }` |
| `DELETE /api/kids/:id` | parent | archive, 204 |
| `POST /api/kids/:id/transactions` | parent | `{ amount: positive int, note: nonempty, direction: 'add'\|'withdraw' }` → txn + new balance |
| `GET /api/kid/summary` | kid | `{ kid: {name}, balance, chart, transactions }` for own kid only |
| `POST /api/test/clock` / `DELETE` | ENABLE_TEST_CLOCK=1 | set/clear fixed time |

Every `/api/kids*` route: 401 no session, 403 kid session. `/api/kid/summary`: 403 for parents.
