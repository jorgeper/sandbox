# Minutes M3 Implementation Plan — Images, Autosave/Recovery, Keep-audio

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** M3 of spec.md — drag-and-drop images into the timeline (live and viewing), 30 s autosave with crash recovery, keep-audio-in-document toggle. (Pause/resume shipped in M1/M2; M3 verifies it end-to-end.)

**Architecture:** Images are *staged*: the drop handler copies nothing — the command records `(id, source_path, wall_time)` in app state and pushes an `Item::Image` into the conversation; bytes are read only when writing the `.mnote`. The UI displays dropped/opened images via the Tauri asset protocol (`convertFileSrc`), with opened `.mnote` assets extracted to an app-data cache dir. Autosave is a thread in the commands layer that snapshots the live conversation JSON to `recovery/current.json` every 30 s plus appends new session audio to `recovery/current.pcm` (raw i16), so a crash loses at most 30 s of audio and no transcript. Settings live in `settings.json` (`keep_audio`, default true).

## Tasks

### Task 1: `.mnote` assets + image items (Rust)
- `write_mnote(path, conv, audio, assets: &[(String, Vec<u8>)])` — assets stored under `assets/` (Stored, they're already compressed). `read_mnote_full(path) -> (Conversation, Vec<(String, Vec<u8>)>)` keeps the simple `read_mnote` for tests.
- Commands: `add_image(path, after_item_id: Option<String>)` — live: appends to engine conversation with `wall_time = now`; stopped/viewing: inserts after the given item (or at end). Stages `(file_name, source_path)` in `Inner`. `save` reads staged bytes + writes assets. `open` extracts assets to `app_data/opened/<conv-id>/` and rewrites nothing (UI resolves `assets/x` → that dir).
- Round-trip test: write with 1 fake image asset + read back both JSON item and asset bytes.

### Task 2: Drag-drop + image rendering (UI)
- `onDragDropEvent` (webview API) → `addImage(path)` for each dropped file with image extension (png/jpeg/gif/webp/heic); non-images ignored with a soft error banner.
- `Item::Image` renders as timeline card via `convertFileSrc` (live drops: original path; opened: extracted path returned by a `asset_base()` command). Mock backend: drop simulation not possible — mock `addImage` pushes an image item with a data-URL placeholder; Playwright asserts an image card renders after calling the mock hook via a test button? No extra UI: Playwright drops are skipped; image rendering is covered by opening the mock conversation containing an image item.
- `assetProtocol` scope enabled for `$APPDATA/**` and `$HOME/**` (dev; tighten in M5 packaging).

### Task 3: Autosave + recovery
- Engine exposes `snapshot() -> (Conversation, new_audio_since_last: Vec<f32>)`; commands autosave thread (spawned on `start_recording`, stopped on `stop`) writes JSON + appends PCM every 30 s (test override env `MINUTES_AUTOSAVE_SECS`).
- On successful `stop`/`save`: recovery files deleted.
- Commands `check_recovery() -> Option<Conversation>`, `recover() -> Conversation` (also re-encodes recovered PCM → ogg into state so Save produces a full `.mnote`), `discard_recovery()`.
- UI: on Home mount, `check_recovery` → banner "Recover last conversation?" [Recover] [Discard].
- Test (Rust): run engine with fake source + `MINUTES_AUTOSAVE_SECS=1`, kill without stop, assert recovery JSON exists and parses with utterances; `recover()` returns them.

### Task 4: Keep-audio setting + pause verification + milestone
- `settings.json` (`keep_audio: bool`, default true), commands `get_settings`/`set_settings`; `save` passes `None` audio when off. Minimal UI: checkbox in the save row ("Keep audio in file"), initialized from settings, persisted on change.
- Pause e2e: engine test — feed fixture, pause mid-stream, feed more (dropped), resume, feed rest; assert t_* continuity (no gap) and transcript only contains non-paused audio.
