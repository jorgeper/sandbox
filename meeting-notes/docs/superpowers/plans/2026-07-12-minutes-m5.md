# Minutes M5 Implementation Plan — Voice Memory, Settings Polish, Packaging

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** M5 of spec.md — opt-in persistent voice library ("Remember this voice"), auto-naming returning voices in new recordings, Voices section in Settings, macOS app bundle build. Notarization (needs Apple Developer credentials) and the Windows MSI (needs a Windows machine) are documented hand-off steps, not automatable here.

**Architecture:** The engine mirrors diarizer centroids into `Shared` after each assignment; `stop()` copies each cluster centroid into `Speaker.embedding` so saved conversations carry voiceprints. `voices.json` (app-data) holds `[{id, name, embedding, created_at, source_conversation}]`. At `Engine::start` the library is loaded; when a **new** cluster is created, its embedding is matched against the library (cosine ≥ 0.70, conservative per spec §5.3) and a hit auto-names the speaker (`auto_named: true` → existing toast+undo UI just works). `remember_voice(speaker_id)` and `forget_voice(voice_id)` are the spec-named commands.

## Tasks

1. **voices.rs + engine wiring** — `VoiceEntry`, `load()/save()/remember()/forget()`; Shared gains `centroids: Mutex<Vec<Vec<f32>>>`; stop() fills `Speaker.embedding`; new-cluster auto-match with library. Tests: voices round-trip; two-session test — session A records alice fixtures, remember her voice, session B (fresh engine) auto-names her "Alice" on first utterance.
2. **Commands + UI** — `list_voices`, `remember_voice`, `forget_voice`. Post-stop speaker cards get "Remember voice" (hidden when already remembered / no embedding); Settings → Voices lists entries with delete. Mock + Playwright coverage.
3. **Packaging + perf + wrap-up** — `npm run tauri build` produces the .app/.dmg (unsigned); confirm the bundled app launches and records. Document notarization + MSI steps in README.md (new). Final full sweep, VERIFY.md, milestone commit, update GOAL status.
