# Minutes M2 Implementation Plan — Diarization + "I am X"

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** M2 of spec.md — per-utterance speaker embeddings (sherpa-onnx), online clustering into Speaker 1/2/3 with palette colors, manual rename, and rule-based "I am X" self-identification with retroactive relabel + undo toast.

**Architecture:** A `diar.rs` module wraps sherpa-rs `EmbeddingExtractor` plus a small cosine-similarity online clusterer. The existing stt worker thread, after transcribing each utterance, computes its embedding and assigns a speaker (single inference thread keeps ordering trivial; the spec's separate-diar-worker split becomes worthwhile only if embedding cost ever matters — it's ~10 ms). A `namer.rs` module holds the pure "I am X" matcher. Renames are cheap because items reference `speaker_id`.

**Tech Stack additions:** sherpa-rs 0.6.8 (default features), WeSpeaker/3D-Speaker English speaker-embedding ONNX model (~30 MB) fetched by the dev script.

## Global Constraints

Same as M1 plan, plus:
- Event `timeline/speaker-updated` payload: full `Speaker` object (camelCase not needed — serde snake_case like document types, matching the TS `Speaker` mirror).
- New command implemented this milestone: `rename_speaker(speaker_id, name)` (manual rename; sets `auto_named: false`).
- "I am X" rules exactly as spec §5.2: patterns `I am X` / `I'm X` / `my name is X` / `this is X`, 1–2 capitalized tokens, stoplist guard, `this is` needs a short utterance (<8 words), manual names never overwritten, undo toast 10 s.
- Speaker colors assigned from `SPEAKER_PALETTE` in creation order, wrapping after 8.
