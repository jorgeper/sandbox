# Minutes M1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** M1 of spec.md — record mic audio, VAD-segment it, transcribe with whisper.cpp in real time, show a live chat-style timeline, and save/open portable `.mnote` documents (single speaker, no diarization yet).

**Architecture:** Tauri 2 app. All audio/inference lives in the Rust core on background threads (capture → channel → VAD segmenter → whisper worker → event emitter). The React webview renders the timeline from Tauri events and drives the engine via Tauri commands. An `AudioSource` trait lets tests substitute a WAV-file source for the microphone.

**Tech Stack:** Tauri 2.11, React 18 + TypeScript + Vite, cpal 0.18, voice_activity_detector 0.2.1 (Silero via ort), whisper-rs 0.16 (Metal), hound 3.5, ogg-opus 0.1.2, zip, crossbeam-channel, serde/serde_json, tauri-plugin-dialog.

## Global Constraints (from spec.md — apply to every task)

- **No network code in the app.** The only allowed network use in v1 is the model downloader (M4). For M1, models are fetched by a **dev shell script**, never by app code.
- Audio pipeline format: **16 kHz mono f32** internally.
- Event names verbatim: `timeline/partial`, `timeline/final`, `timeline/speaker-updated`, `engine/status`, `audio/level`, `models/download-progress`.
- Command names verbatim: `start_recording`, `pause`, `resume`, `stop`, `save`, `open`, `rename_speaker`, `delete_item`, `add_image`, `download_model`, `delete_model`, `remember_voice`, `forget_voice` (M1 implements only: `start_recording`, `pause`, `resume`, `stop`, `save`, `open`, `new_conversation`).
- `.mnote` = zip: `conversation.json` (+ `audio.ogg` Opus 24 kbps mono, `assets/`), schema exactly as spec §4.2, `schema_version: 1`.
- Utterance boundary: ≥600 ms silence ends an utterance; force-split at 30 s. Partial transcription pass ~every 1.5 s.
- UI: system font stack, one accent color (`#5B8DEF`), light/dark follows OS, no component framework.
- macOS `NSMicrophoneUsageDescription` required. App identifier: `com.jorgeper.minutes`.
- Latency target: final text ≤ ~2 s after end of utterance with `small` model on this Mac.
- Commit after every green task. Conventional-commit style, scope `meeting-notes` (e.g. `feat(meeting-notes): ...`).

### Task 6: Engine — threads, state machine, events

**Files:**
- Create: `src-tauri/src/engine.rs`; register `mod engine;`
- Test: inline `#[cfg(test)]` using `WavFileSource`

**Interfaces:**
- Consumes: `AudioSource`, `Segmenter`, `Transcriber`, `Conversation`.
- Produces (used by commands.rs):
  - `enum EngineEvent { Partial { utterance_id: String, text: String }, Final(crate::document::Item), Level { rms: f32 }, Status { state: String } }` — serde-serializable, camelCase payload fields to match TS
  - `struct Engine` with:
    - `Engine::start(source: Box<dyn AudioSource>, model: &str, events: Sender<EngineEvent>) -> anyhow::Result<Engine>` — spawns segmenter thread + stt thread; emits `Status{recording}`
    - `fn pause(&self)` / `fn resume(&self)` (atomic flag; paused chunks are dropped before the segmenter and before the session buffer, so `t_*` and audio stay pause-adjusted; emits status)
    - `fn stop(self) -> anyhow::Result<(crate::document::Conversation, Vec<f32>)>` — flushes segmenter, waits for stt queue to drain, returns finished conversation (items filled, `ended_at` set) + full 16 kHz session audio
  - Utterance ids `utt-0001…`; every final utterance gets `speaker_id` of the single "Speaker 1", `wall_time = started_at + t_start-with-pause-offsets` (M1 simplification: `wall_time = Local::now()` at finalization is acceptable and monotone)
  - Partial pass: while an utterance is open and the stt worker is idle, every ≥1.5 s transcribe the open utterance's samples-so-far and emit `Partial` with the *upcoming* utterance id; skip when a final job is queued.
  - `Level` events at ~15 Hz (RMS over each incoming chunk batch).
