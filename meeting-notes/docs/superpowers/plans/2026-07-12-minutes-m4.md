# Minutes M4 Implementation Plan — OOBE + Model Manager

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** M4 of spec.md — first-launch OOBE wizard (welcome → mic permission → model choice → verified download → done) and a Settings surface with the model manager. The in-app downloader is the app's **only** network code, ever, and runs solely on explicit user action.

**Architecture:** `models.rs` owns a static catalog (name, engine, size, sha256 — hashes pinned from the HF API / local verification on 2026-07-12) and a blocking `ureq` downloader that streams to `<file>.part`, emits `models/download-progress` events, sha256-verifies, then renames into place. Settings grow `stt_model` and `oobe_done` (serde-defaulted so existing files parse). `start_recording` reads the active model from settings. The UI gains an OOBE wizard (shown when `oobe_done` is false) and a Settings panel (model list with download/delete, active-model picker, keep-audio, re-run OOBE).

## Tasks

1. **models.rs + commands** — catalog (whisper tiny/base/small/large-v3-turbo + speaker-embedding CAM++), `list_models() -> Vec<ModelInfo{name, kind, size_bytes, installed, active}>`, `download_model(name)` (thread + progress events + sha256 + `.part` rename; error event on mismatch), `delete_model(name)` (refuses while recording). Settings: `stt_model`, `oobe_done` with `#[serde(default)]`. `start_recording` uses `settings.stt_model`. Unit tests: catalog integrity (names unique, hashes 64 hex chars), sha256 verify function against a fixture byte string, settings round-trip with old-format JSON missing new fields.
2. **request_mic_permission command** — briefly opens and closes a `MicSource` so the macOS TCC prompt fires from the wizard; returns granted/denied best-effort (cpal error ⇒ denied/unavailable).
3. **OOBE UI** — 5 steps (welcome+local promise / mic / transcription model table with size-speed-quality and `small` preselected / download with per-model progress bars, "only time Minutes uses the network" note / done → sets `oobe_done`). Shown on launch when `oobe_done` is false; re-runnable from Settings. Skippable download leaves viewer-only mode (record button shows "models needed" CTA).
4. **Settings UI** — gear from Home: Models section (installed/available, active picker, download/delete), keep-audio toggle (moves here, stays synced with the save-row checkbox), "Re-run setup". Mock backend scripts catalog + fake progress; Playwright: OOBE walk-through (choose base → progress reaches 100% → done lands Home) and Settings delete/download states.
5. **Verify + milestone** — real download of `tiny` through the app command (delete after), sha256 mismatch path unit-tested, full sweep, VERIFY.md, commit.
